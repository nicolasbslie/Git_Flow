# Git_Flow

--- 

1. Criar um novo respositório
2. Garantir que ele já tenha main e develop
3. Clonar o repositório
4. Criar 3 funcionalidades (features) diferentes
5. Sigam o passo a passo do gitflow para cada uma delas
6. Criar a release
7. Simular mudança e/ou correção de código na release
8. Enviar para o github remoto e fazer o merge com as branchs necessárias
9. Criar o hotfix e simular a correção de um erro na main
10. Enviar também para o github remoto e fazer o merge com as branchs necessárias
